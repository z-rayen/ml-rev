Vue d'ensemble — Systèmes de Recommandation

Qu'est-ce qu'un système de recommandation ? DÉFINITION

Un système de recommandation prédit ce qu'un utilisateur aimerait voir ou acheter, en se basant sur ses préférences passées et/ou sur celles d'autres utilisateurs similaires.

Exemples réels : Netflix ("Films recommandés pour toi"), Amazon ("Les clients ont aussi acheté"), Spotify ("Discover Weekly"), YouTube (suggestions de vidéos).
📈
Simple / Popularity

Recommande les items les plus populaires. Ne personnalise pas. Formule IMDB.

🎬
Content-Based

Recommande des items similaires à ce que l'utilisateur a aimé. Basé sur les attributs des items.

👥
Collaborative Filtering

Recommande en se basant sur des utilisateurs similaires. "Les gens comme toi ont aimé..."

01 📈

Simple Recommender — Formule IMDB

Problème avec la moyenne brute PIÈGE EXAMEN

Un film avec 5 votes à 10/10 aurait un score parfait, alors qu'un film avec 10 000 votes à 8.5/10 est probablement bien meilleur. La moyenne brute est trompeuse.

Solution : La formule IMDB pondère le score par rapport au nombre de votes et à la moyenne globale.
📐 Formule IMDB Weighted Rating
score = (v / (v + m)) × R + (m / (v + m)) × C
  • v = vote_count du film
  • m = seuil minimum de votes (ex: quantile 70%)
  • R = vote_average du film
  • C = moyenne globale de tous les films
Intuition : Si v est petit → le score tend vers C (la moyenne globale). Si v est grand → le score tend vers R (le vrai score du film). Le film doit "mériter" son score avec suffisamment de votes.
PYTHON — SIMPLE RECOMMENDER (IMDB)
# 1. Filtrer les films trop courts ou trop longs data = data.loc[(data['runtime'] >= 50) & (data['runtime'] < 300)] # 2. Seuil minimum de votes (quantile 70%) → exclure les films peu connus m = data['vote_count'].quantile(0.7) data = data.loc[data['vote_count'] > m] # 3. Moyenne globale de vote_average C = data['vote_average'].mean() # 4. Définir la fonction IMDB def IMDB_score(data, m=m, C=C): R = data.vote_average v = data.vote_count return ((v / (v + m)) * R) + ((m / (v + m)) * C) # 5. Appliquer et trier data['score_imdb'] = data.apply(IMDB_score, axis=1) data.sort_values(by='score_imdb', ascending=False).head(10)

Content-Based Filtering

Principe CONCEPT CLÉ

On crée un profil utilisateur à partir des items qu'il a aimés. Ensuite, on recommande les items les plus similaires à ce profil.

Items regardés
→
Encodage (genre/texte)
→
User Profile (moyenne)
→
Cosine Similarity
→
Top-N recommandations
🏷️ MultiLabelBinarizer OUTIL

Transforme une liste de genres en vecteur binaire one-hot.

# Entrée : ['Action', 'Comedy'] # Sortie : [1, 0, 1, 0, 0, ...] from sklearn.preprocessing import MultiLabelBinarizer mlb = MultiLabelBinarizer() # fit_transform sur la liste de genres genres_onehot = mlb.fit_transform(data['genre_names'])
⚙️ Pipeline Genres
  1. Extraire genres depuis JSON : eval(x)
  2. Extraire noms : [d['name'] for d in x]
  3. Nettoyer (retirer studios)
  4. MultiLabelBinarizer → one-hot
  5. Créer user profile (mean)
  6. Cosine similarity → tri
PYTHON — CONTENT-BASED (GENRES)
# ── Extraction des genres ────────────────────────── data['genre_list'] = data['genres'].apply(lambda x: eval(x)) data['genre_names'] = data['genre_list'].apply(lambda x: [d['name'] for d in x]) # ── Nettoyage : supprimer les studios mal classifiés ── bad = ['The Cartel', 'Aniplex', 'GoHands'] # etc. data['genre_names'] = data['genre_names'].apply( lambda x: [g for g in x if g not in bad]) # ── Encodage one-hot ──────────────────────────────── mlb = MultiLabelBinarizer() genres_onehot = mlb.fit_transform(data['genre_names']) # ── Films vus par l'utilisateur ───────────────────── user_watched = ['49026', '99861', '293660'] user_watch_list = data.loc[data['id'].isin(user_watched)] # ── Profil utilisateur = moyenne des genres vus ────── user_profile = user_watch_list[mlb.classes_].mean() user_profile = user_profile.values.reshape(1, -1) # ── Cosine Similarity ─────────────────────────────── from sklearn.metrics.pairwise import cosine_similarity cosine_score = cosine_similarity(genres_onehot, user_profile) data['score'] = cosine_score.flatten() # ── Top 10 recommandations (exclure films déjà vus) ── recs = data.loc[~data['id'].isin(user_watched)] recs.sort_values('score', ascending=False)[['title', 'score']].head(10)
⚠️ Limite du genre seul : Deux films d'action peuvent être très différents. Le Dark Knight (super-héros, sombre) vs Fast & Furious (voitures, action). Les genres seuls ne suffisent pas → utiliser aussi le résumé (overview).
03 📝

TF-IDF — Encoder le texte

Pourquoi TF-IDF ? CONCEPT

TF-IDF transforme un texte (overview de film) en vecteur numérique. Les mots importants (rares dans le corpus mais fréquents dans le document) ont un poids élevé. Les mots courants ("the", "a", "is") sont ignorés.

📐 TF — Term Frequency
TF(t,d) = nb occurrences de t dans d / nb total de mots dans d

Fréquence du mot dans le document. Un mot qui apparaît souvent est important pour ce document.

📐 IDF — Inverse Document Frequency
IDF(t) = log(N / df(t))

N = nb total de documents. df(t) = nb de documents qui contiennent t. Un mot présent partout (faible IDF) est peu informatif.

💡 Intuition avec exemple
MOT : "Batman"
Apparaît dans 3 films sur 10 000. IDF élevé → mot très discriminant → poids fort.
MOT : "movie"
Apparaît dans 9 800 films. IDF très faible → mot non informatif → poids faible.
PYTHON — TF-IDF SUR LES OVERVIEWS
from sklearn.feature_extraction.text import TfidfVectorizer # Remplir les NaN par une chaîne vide movies_overview = data.overview.fillna('').values # Créer le vectoriseur # stop_words : ignore les mots courants anglais # max_features : garde seulement les 5000 mots les plus importants vectorizer = TfidfVectorizer(stop_words='english', max_features=5000) # fit_transform sur TOUS les films tfidf_matrix = vectorizer.fit_transform(movies_overview).toarray() # shape : (nb_films, 5000) # Pour les films de l'utilisateur → transform() seulement (pas fit !) user_tfidf = vectorizer.transform(user_watch_list['overview'].values).toarray() # ⚠️ IMPORTANT : fit sur tout le corpus, transform sur les sous-ensembles
⚠️ fit() vs transform() : On fait fit_transform() sur l'ensemble complet des films. Ensuite, pour les films de l'utilisateur, on fait seulement transform() — jamais fit_transform() — pour utiliser le même vocabulaire.

User Profile — Construction

Qu'est-ce qu'un User Profile ? FONDAMENTAL

Le profil utilisateur est un vecteur numérique qui résume les préférences de l'utilisateur. Il est calculé comme la moyenne des vecteurs des items qu'il a aimés.

user_profile = mean(vecteur_film_1, vecteur_film_2, ..., vecteur_film_n)
CONSTRUCTION DU PROFIL UTILISATEUR
Film 1 (Dark Knight)
[0,1,1,0,1,0,...]
+
Film 2 (Avengers)
[0,1,1,1,0,0,...]
+
Film 3 (Deadpool)
[1,1,0,1,0,0,...]
÷3 =
User Profile
[0.33,1,0.67,0.67,...]
PYTHON — CONSTRUCTION DU USER PROFILE
# Films vus par l'utilisateur user_watched = ['49026', '99861', '293660'] user_watch_list = data.loc[data['id'].isin(user_watched)] # ── Cas 1 : User Profile via genres (DataFrame) ────── user_profile = user_watch_list[genres_names].mean() user_profile = user_profile.values.reshape(1, -1) # (1, n_genres) # ── Cas 2 : User Profile via TF-IDF (numpy) ────────── user_tfidf = vectorizer.transform(user_watch_list['overview'].values).toarray() user_profile = user_tfidf.mean(axis=0) # moyenne sur les lignes user_profile = user_profile.reshape(1, -1) # (1, 5000) # ── Cosine Similarity ──────────────────────────────── cosine_score = cosine_similarity(tfidf_matrix, user_profile) data['score'] = cosine_score.flatten()
05 🔀

Combinaison Multi-Features

Pourquoi combiner ? CONCEPT

Utiliser un seul type de feature (genre seul, ou texte seul) donne des résultats imparfaits. On peut combiner genres + overview + type + popularité pour des recommandations plus précises.

⚡ Early Fusion CONCATENATION

Encoder séparément chaque feature, puis concaténer tous les vecteurs en un seul grand vecteur. Un seul calcul de cosine similarity.

content_vector = [genre_onehot | type_onehot | rating_scaled | members_scaled]
Challenge : Une feature avec beaucoup de dimensions peut dominer les autres.
🔮 Late Fusion PONDÉRATION

Calculer un cosine similarity séparé pour chaque feature, puis combiner avec des poids.

score = 0.4×cos(genre) + 0.2×cos(type) + 0.2×cos(members) + 0.2×cos(ratings)
Avantage : Contrôle fin du poids de chaque feature. Challenge : Comment trouver les bons poids ?
PYTHON — EARLY FUSION (np.concatenate)
from sklearn.preprocessing import OneHotEncoder, MinMaxScaler, Normalizer # Encoder le type (Movie, TV Series, etc.) ohe = OneHotEncoder() type_onehot = ohe.fit_transform(data[['type']]).toarray() # Scaler les features numériques rating_scaled = MinMaxScaler().fit_transform(data[['rating']]) members_scaled = MinMaxScaler().fit_transform(np.log1p(data[['members']])) # np.log1p : log(1+x) pour réduire l'effet des grands nombres # Normaliser avant concatenation (optionnel mais recommandé) genres_norm = Normalizer().fit_transform(genres_onehot) tfidf_norm = Normalizer().fit_transform(tfidf_matrix) # Concatener tous les vecteurs content_vector = np.concatenate([ genres_norm, # one-hot genres type_onehot, # one-hot type rating_scaled, # score normalisé members_scaled # popularité normalisée ], axis=1) print(content_vector.shape) # (nb_films, somme des dims)
💡 np.log1p(members) : La popularité peut varier de 1 à 1 000 000. Sans transformation, elle dominerait tout. log(1+x) compresse ces valeurs dans une plage raisonnable.
06 👥

Collaborative Filtering

Principe CONCEPT CLÉ

Au lieu d'analyser le contenu des items, le CF analyse les comportements des utilisateurs. Si l'utilisateur A et l'utilisateur B ont des goûts similaires, ce que B a aimé peut être recommandé à A.

Analogie : "Les amis qui ont des goûts similaires aux tiens ont aussi aimé ce film."
👤 User-Based CF

Trouver des utilisateurs similaires à l'utilisateur cible, puis recommander ce qu'ils ont aimé.

User A ↔ User B ← similaires
B a aimé film X
→ Recommander X à A
🎬 Item-Based CF

Trouver des items similaires à ce que l'utilisateur a aimé, en se basant sur les patterns de notation.

Film X ↔ Film Y ← souvent notés ensemble
User A a aimé X
→ Recommander Y à A
📊 User-Item Matrix — La structure centrale

Le point de départ du CF : une matrice où les lignes = utilisateurs, les colonnes = items, et les valeurs = notes. Les cases vides (0 ou NaN) = non noté.

# Construire la User-Item Matrix user_item_matrix = pd.pivot_table( data, index='userId', columns='movieId', values='rating' ) user_item_matrix.fillna(0, inplace=True) # Shape : (nb_users, nb_movies)
Résultat : Une matrice (nb_users × nb_movies). La plupart des cases sont 0 car les utilisateurs n'ont pas noté la majorité des films → matrice sparse (creuse).
PYTHON — USER-BASED COLLABORATIVE FILTERING
from sklearn.metrics.pairwise import cosine_similarity # ── Similarité entre utilisateurs ─────────────────── # On transpose : chaque utilisateur devient une colonne user_similarity = cosine_similarity(user_item_matrix.T) user_similarity_df = pd.DataFrame( user_similarity, index=user_item_matrix.index, columns=user_item_matrix.index ) # ── Prédiction des notes ───────────────────────────── # Formule : note prédite = Σ(sim × notes) / Σ(|sim|) def predict_ratings(similarity, ratings_matrix): sim = similarity ratings = ratings_matrix.values predicted = sim.dot(ratings) / np.abs(sim).sum(axis=0).reshape(-1, 1) return pd.DataFrame(predicted, index=ratings_matrix.index, columns=ratings_matrix.columns) predictions = predict_ratings(user_similarity, user_item_matrix) # ── Recommandation pour un utilisateur ────────────── def recommend_for_user(user_id, predictions, original_data): # Films déjà notés (à exclure) already_rated = original_data.loc[user_id] already_rated = already_rated[already_rated > 0].index user_pred = predictions.loc[user_id] user_pred = user_pred.drop(already_rated) return user_pred.sort_values(ascending=False).head(10) recs = recommend_for_user(4, predictions, user_item_matrix)
⚠️ Problèmes du Collaborative Filtering
Cold Start Problem

Un nouvel utilisateur (0 notes) ou un nouvel item (0 votes) → impossible de calculer la similarité.

Sparsity (Matrice creuse)

La plupart des utilisateurs n'ont noté qu'un petit pourcentage des items. La matrice est quasi-vide.

Cosine Similarity — Le moteur des RS

Définition FORMULE EXAMEN

La similarité cosinus mesure le cosinus de l'angle entre deux vecteurs. Elle varie de -1 à 1 (ou 0 à 1 pour des vecteurs non-négatifs comme les one-hot).

cos(A,B) = (A · B) / (||A|| × ||B||)
+1

Même direction. Identiques.
Angle = 0°

0

Orthogonaux. Aucune similarité.
Angle = 90°

-1

Directions opposées. Contraires.
Angle = 180°

PYTHON — COSINE SIMILARITY (TOUS LES CAS)
from sklearn.metrics.pairwise import cosine_similarity import numpy as np # ── Exemple simple ─────────────────────────────────── A = np.array([[1, 2, 3]]) B = np.array([[1, 2, 3]]) # → cos = 1.0 (identiques) C = np.array([[-1, -2, -3]]) # → cos = -1.0 (opposés) # ── User Profile vs tous les items ─────────────────── # user_profile shape : (1, n_features) # items_matrix shape : (n_items, n_features) scores = cosine_similarity(items_matrix, user_profile) # scores shape : (n_items, 1) data['score'] = scores.flatten() # .flatten() → 1D array # ── Matrice de similarité utilisateurs ─────────────── # user_item_matrix shape : (n_users, n_items) # .T = transpose pour avoir les users en colonnes user_sim = cosine_similarity(user_item_matrix.T) # user_sim shape : (n_users, n_users)
💡 .flatten() : cosine_similarity retourne une matrice 2D (n_items, 1). On utilise .flatten() pour obtenir un array 1D qu'on peut assigner comme colonne d'un DataFrame.

# ════════════════════════════════════════════════════ # CONTENT-BASED : Genre + Overview (notebook combiné) # ════════════════════════════════════════════════════ import pandas as pd, numpy as np from sklearn.preprocessing import MultiLabelBinarizer, Normalizer from sklearn.feature_extraction.text import TfidfVectorizer from sklearn.metrics.pairwise import cosine_similarity data = pd.read_csv('movies_metadata.csv', low_memory=False) # ── 1. GENRES ───────────────────────────────────────── data['genre_names'] = data['genres'].apply( lambda x: [d['name'] for d in eval(x)]) # Nettoyage : enlever les non-genres mlb = MultiLabelBinarizer() genres_onehot = mlb.fit_transform(data['genre_names']) genres_onehot = Normalizer().fit_transform(genres_onehot) # ── 2. OVERVIEW (TF-IDF) ────────────────────────────── vectorizer = TfidfVectorizer(stop_words='english', max_features=5000) tfidf = vectorizer.fit_transform(data.overview.fillna('')).toarray() tfidf = Normalizer().fit_transform(tfidf) # ── 3. CONCATENER ───────────────────────────────────── encoding = np.concatenate([tfidf, genres_onehot], axis=1) # ── 4. RECOMMANDATION ──────────────────────────────── data['score'] = cosine_similarity(encoding, user_profile).flatten() recs = data.loc[~data['id'].isin(user_watched)] recs.sort_values('score', ascending=False)[['title', 'score']].head(10)

🎯 Points à ne pas confondre à l'examen
fit_transform() vs transform()
fit_transform() = apprendre + transformer (sur le train/corpus complet)
transform() = transformer seulement (sur les nouvelles données)
mean(axis=0) vs mean()
mean(axis=0) = moyenne sur les lignes → vecteur profil
mean() sur un DataFrame = même chose via pandas
reshape(1,-1)
cosine_similarity attend une matrice 2D. Un vecteur 1D doit être reshape(1,-1) avant d'être passé.
~isin() pour exclure
~ = NOT en pandas. Exclure les films déjà vus :
