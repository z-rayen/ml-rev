Apprentissage Non Supervisé

Définition FONDAMENTAL

L'apprentissage non supervisé est une branche du Machine Learning où l'algorithme apprend à partir de données sans étiquettes (labels). L'objectif est de découvrir une structure cachée dans les données.

Analogie : Imagine un tas de pièces de monnaie de différents pays. Tu n'as pas de liste qui dit "celle-ci est française, celle-là est américaine". Mais tu peux les regrouper par taille, couleur et poids. C'est exactement le clustering.
✅ Supervisé
  • Données avec labels (y)
  • Classification, Régression
  • Évalue sur train/test split
  • Ex: prédire si email = spam
🔍 Non Supervisé
  • Données sans labels
  • Clustering, Réduction de dims
  • Pas de "bonne réponse"
  • Ex: segmenter des clients
🗺️ Vue d'ensemble du pipeline
Données brutes → Nettoyage + Scaling → PCA (visualisation) → KMeans / DBSCAN → Évaluation
01 ⚙️

KMeans Clustering

Comment ça marche ? ALGO

KMeans divise les données en K groupes (clusters) en minimisant la distance entre chaque point et son centroïde (centre du cluster).

  1. Initialisation : Choisir K centroïdes aléatoirement (ou avec KMeans++)
  2. Attribution : Assigner chaque point au centroïde le plus proche (distance euclidienne)
  3. Mise à jour : Déplacer chaque centroïde vers la moyenne de ses points
  4. Répéter les étapes 2-3 jusqu'à convergence (les centroïdes ne bougent plus)
📐 Inertie

Somme des distances au carré de chaque point à son centroïde. C'est la fonction à minimiser.

Inertie = Σ ||xᵢ − μₖ||²

μₖ = centroïde du cluster k

⚠️ Limites
  • Il faut choisir K à l'avance
  • Sensible aux outliers
  • Suppose des clusters sphériques
  • Résultat peut varier (random init)
Conseil : Toujours utiliser n_init=10 et random_state=42 pour la reproductibilité.
PYTHON — KMEANS COMPLET
from sklearn.cluster import KMeans from sklearn.metrics import silhouette_score # Appliquer KMeans avec K=3 kmeans = KMeans(n_clusters=3, random_state=42, n_init=10) kmeans_labels = kmeans.fit_predict(x_scaled) # Évaluer print(f"Inertie: {kmeans.inertia_:.2f}") print(f"Silhouette: {silhouette_score(x_scaled, kmeans_labels):.4f}") # Accéder aux centroïdes centroids = kmeans.cluster_centers_ # shape: (K, n_features)
💡 À l'examen : KMeans est sensible à l'échelle des features. C'est pourquoi on applique StandardScaler AVANT. Sans scaling, une feature avec de grandes valeurs (ex: balance en milliers) dominera le calcul des distances.
02 🔵

DBSCAN

Density-Based Spatial Clustering DENSITÉ

DBSCAN groupe les points qui sont densément regroupés. Contrairement à KMeans, il trouve automatiquement le nombre de clusters et peut identifier les outliers (bruit).

🔧 Paramètres clés
eps rayon de voisinage

Distance maximale entre deux points pour être "voisins". Trop petit → tout est bruit. Trop grand → tout est un seul cluster.

min_samples densité minimale

Nombre min de points dans le rayon eps pour former un cluster. Règle : min_samples = 2 × n_features

🏷️ Types de points
  • Core point : a ≥ min_samples voisins dans eps
  • Border point : dans le voisinage d'un core point, mais pas core lui-même
  • Noise point : label -1, outlier
Label -1 = BRUIT. Ces points ne sont dans aucun cluster.
🔍 Trouver les meilleurs paramètres
📈
K-Distance Graph

Trouver le "coude" → valeur eps

📊
Silhouette Score

Tester plusieurs eps, garder le meilleur score

🔲
Grid Search

Tester toutes les combinaisons eps × min_samples

PYTHON — DBSCAN COMPLET
from sklearn.cluster import DBSCAN # Règle du pouce : min_samples = 2 × n_features min_samples = 2 * x_scaled.shape[1] # 2 × 17 = 34 # Appliquer DBSCAN dbscan = DBSCAN(eps=2.5, min_samples=min_samples) dbscan_labels = dbscan.fit_predict(x_scaled) # Compter clusters et bruit n_clusters = len(set(dbscan_labels)) - (1 if -1 in dbscan_labels else 0) n_noise = list(dbscan_labels).count(-1) print(f"Clusters: {n_clusters}, Bruit: {n_noise}") # ⚠️ Silhouette nécessite >= 2 clusters ! if n_clusters >= 2: score = silhouette_score(x_scaled, dbscan_labels) print(f"Silhouette: {score:.4f}")
⚠️ Piège fréquent : Le silhouette score ne peut être calculé que si DBSCAN trouve au moins 2 clusters et que tous les points ne sont pas bruit. Toujours vérifier avant de calculer !
03 📐

Elbow Method

Principe CHOIX DE K

L'inertie diminue toujours quand K augmente (plus de clusters = chaque point est plus proche d'un centroïde). On cherche le point où l'amélioration ralentit significativement — le "coude".

COURBE D'INERTIE — MÉTHODE DU COUDE
PYTHON — ELBOW METHOD
k_range = range(2, 11) inertia = [] for k in k_range: kmeans = KMeans(n_clusters=k, random_state=42, n_init=10) kmeans.fit(x_scaled) inertia.append(kmeans.inertia_) plt.plot(k_range, inertia, marker='o') plt.xlabel('Nombre de clusters (K)') plt.ylabel('Inertie') plt.title('Elbow Method') plt.show()
💡 Note : Le coude n'est pas toujours évident. Dans ce cas, combiner avec le silhouette score pour prendre la meilleure décision.
04 📊

Silhouette Score

Définition ÉVALUATION

Le silhouette score mesure à quel point un point est bien regroupé avec son cluster par rapport aux clusters voisins. Il varie de -1 à +1.

s(i) = (b(i) − a(i)) / max(a(i), b(i))

a(i) = distance moyenne entre i et les autres points de son cluster (cohésion)

b(i) = distance moyenne minimale entre i et les points d'un autre cluster (séparation)

≈ +1

Point bien dans son cluster, loin des autres. Excellent

≈ 0

Point sur la frontière entre deux clusters. Ambigu

≈ -1

Point mal classifié, devrait être dans un autre cluster. Mauvais

PYTHON — SILHOUETTE POUR KMEANS
scores = [] silhouette_range = range(2, 11) for k in silhouette_range: kmeans = KMeans(n_clusters=k, random_state=42, n_init=10) labels = kmeans.fit_predict(x_scaled) scores.append(silhouette_score(x_scaled, labels)) # Trouver le meilleur K best_k = list(silhouette_range)[np.argmax(scores)] print(f"Meilleur K: {best_k} (score = {max(scores):.4f})")
Silhouette pour DBSCAN : Même logique, mais attention : calculer seulement si n_clusters >= 2. Les points bruit (label -1) sont inclus dans le calcul.
05 🔻

Réduction de Dimensions · PCA

Principal Component Analysis VISUALISATION

Notre dataset a 17 features (17 dimensions). Il est impossible de visualiser 17 dimensions. PCA réduit à 2D ou 3D en conservant le maximum d'information.

Analogie : Projeter une sphère 3D sur un mur. Tu perds une dimension mais tu gardes la forme générale — son "ombre" 2D. PCA trouve la meilleure projection possible.
📈 Variance expliquée

Chaque composante principale capture une portion de la variance totale :

PC1 (1ère composante)~26%
PC2 (2ème composante)~14%
Total (PC1+PC2)~40%

On perd ~60% d'info. C'est normal pour la visualisation uniquement — on ne clustering pas sur PCA.

⚠️ PCA vs Clustering
  • On scale d'abord, puis PCA
  • PCA sert à visualiser, pas à clusterer
  • Le clustering se fait sur x_scaled (17D)
  • On projette ensuite les résultats sur PCA pour afficher
kmeans.fit(x_scaled) ← correct
kmeans.fit(x_pca) ← perte d'info !
PYTHON — PCA COMPLET
from sklearn.decomposition import PCA # Réduire à 2 dimensions pour visualisation pca = PCA(n_components=2) x_pca = pca.fit_transform(x_scaled) # Variance expliquée print(f"PC1: {pca.explained_variance_ratio_[0]*100:.1f}%") print(f"PC2: {pca.explained_variance_ratio_[1]*100:.1f}%") print(f"Total: {sum(pca.explained_variance_ratio_)*100:.1f}%") # Projeter les centroïdes KMeans sur PCA (pour affichage) centroids_pca = pca.transform(kmeans.cluster_centers_)
06 ⚖️

KMeans vs DBSCAN

CRITÈRE KMEANS DBSCAN
Nb de clusters ⚠️ À choisir manuellement (K) ✅ Découvert automatiquement
Forme des clusters ⚠️ Assume des clusters sphériques ✅ N'importe quelle forme
Outliers / Bruit ❌ Tout point assigné à un cluster ✅ Points bruit = label -1
Paramètres K (nombre de clusters) eps + min_samples
Scalabilité ✅ Très rapide sur grands datasets ⚠️ Plus lent sur très grands datasets
Idéal pour Clusters bien séparés et ronds Données avec bruit, formes irrégulières
Reproductibilité ⚠️ Dépend de l'init (random_state) ✅ Déterministe
💡 Règle d'or à l'examen : Il n'existe pas de "meilleur" algorithme universel. Le choix dépend des données. Si tu vois des outliers ou des formes irrégulières → DBSCAN. Si les clusters sont compacts et ronds → KMeans.

PIPELINE COMPLET DU DS
# ──── 1. IMPORTS ──────────────────────────────── import pandas as pd import numpy as np import matplotlib.pyplot as plt from sklearn.preprocessing import StandardScaler from sklearn.decomposition import PCA from sklearn.cluster import KMeans, DBSCAN from sklearn.metrics import silhouette_score # ──── 2. CHARGEMENT ET NETTOYAGE ───────────────── data = pd.read_csv('CC GENERAL.csv') data['MINIMUM_PAYMENTS'] = data['MINIMUM_PAYMENTS'].fillna( data['MINIMUM_PAYMENTS'].median()) data['CREDIT_LIMIT'] = data['CREDIT_LIMIT'].fillna( data['CREDIT_LIMIT'].median()) # ──── 3. FEATURES + SCALING ────────────────────── x = data.drop(columns=['CUST_ID']) # supprimer colonne texte scaler = StandardScaler() x_scaled = scaler.fit_transform(x) # mean=0, std=1 # ──── 4. PCA (visualisation seulement) ─────────── pca = PCA(n_components=2) x_pca = pca.fit_transform(x_scaled) # ──── 5. ELBOW + SILHOUETTE (trouver K) ────────── k_range = range(2, 11) inertia, sil_scores = [], [] for k in k_range: km = KMeans(n_clusters=k, random_state=42, n_init=10) labels = km.fit_predict(x_scaled) inertia.append(km.inertia_) sil_scores.append(silhouette_score(x_scaled, labels)) best_k = list(k_range)[np.argmax(sil_scores)] # ──── 6. KMEANS FINAL ──────────────────────────── kmeans = KMeans(n_clusters=best_k, random_state=42, n_init=10) kmeans_labels = kmeans.fit_predict(x_scaled) # ──── 7. DBSCAN ────────────────────────────────── min_samples = 2 * x_scaled.shape[1] # 34 dbscan = DBSCAN(eps=2.5, min_samples=min_samples) dbscan_labels = dbscan.fit_predict(x_scaled) n_clusters = len(set(dbscan_labels)) - (1 if -1 in dbscan_labels else 0)
🎯 Points essentiels à mémoriser

StandardScaler

mean=0, std=1. Obligatoire avant clustering car KMeans est sensible aux distances.

Silhouette Score

Entre -1 et +1. Plus c'est proche de 1, mieux les clusters sont séparés.

Elbow Method

L'inertie ↓ quand K ↑. On cherche le "coude" = point d'inflexion.

DBSCAN label -1

Points bruit = outliers. Pas dans un cluster. Unique à DBSCAN.
